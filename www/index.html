Объективная оценка качества


Централизация данных

CallAider собирает и хранит информацию о реализации каждого сервиса элементами сети оператора

(SMSC, коммутаторами, HLR, билингом, CAMEL-сервисами и даже базовыми станциями) используя высокопроизводительные анализаторы сигнального и IP трафика.
Полный анализ событий

CallAider анализирует всю последовательность событий, результат каждого из них на каждом этапе выполнения сервиса и контролирует соответствие параметров критериям качества.

Своевременные и полные отчеты

Вы можете мгновенно получить отчет по KPI сервиса и показателям нагрузки, или периодически получать отчеты по каждому сервису.






Сокращение издержек на техническое обслуживание

Детальная информация о каждом вызове

Инженеры службы поддержки могут оперативно и без привлечения профильных специалистов получить полную последовательность событий для каждого вызова, для каждого случая обращения к сервису и самостоятельно локализовать проблему, с помощью удобного интерфейса, без разбора журналов и сетевого трафика.

Своевременная информационная поддержка

CallAider хранит всю информацию, необходимую для разбора заявок о работе сервиса в прошлом. Специалисты поддержки всегда смогут определить причину. Пользователи CallAider уже забыли про ожидание повторной аварии для выяснения причин.

Безопасность персональных данных

Службе поддержки предоставляется отдельный интерфейс, который гарантирует безопасность личной информации абонентов

Автоматические сценарии решения проблемы

CallAider поддерживает сценарии, которые выполняются при определенном одиночном событии или последовательности событий. Например: если абонент более 5 раз столкнулся с ошибкой при handover, формируется заявка в техническую поддержку, абоненту отправляется SMS, что его проблема выялена и передана для решения в службу поддержки.

Статические отчеты и оперативный мониторинг


Анализ индикаторов качества сервиса

При обнаружении существенных отклонений в качестве сервиса, специалисты службы поддержки легко могут провести анализ зависимости частоты возникновения ошибок от режима работы, моделей мобильного терминала и других параметров сервиса. Такое разделение поможет выявить истинную причину отклонения, например, выход новой версии iOS, реконфигурация одного из коммутаторов или “зависание” одного из сигнальных линков.

Мгновенная реакция на изменения

Определение ключевых показателей качества сервиса и их нормальных значений используется для включения в общую систему мониторинга. Службы мониторинга незамедлительно информируются о возникновении чрезвычайных ситуаций и существенной деградации сервиса. Мгновенная реакция - ваша защита от серьёзных последствий.

Эффективное использование опыта

Каждая “закрытая” заявка по качеству работы сервиса может использоваться как фильтр, обнаружения повторного возникновения проблемы. CallAider автоматически “откроет” заявку при ее повторном воспроизведении и оповестит специалистов поддержки.

Автоматическое выявление тенденций

Накопленная статистика эффективно используется для обнаружения тенденций ухудшения качества сервиса, выявления причин этих изменений и принятия привентивных мер.
